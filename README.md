# TorrentSide

`TorrentSide` — это переработанный private tracker на базе старого TBDev-стека, адаптированный под актуальный PHP/MySQL, Docker-окружение и современную логику работы трекера. Репозиторий уже содержит не только классический листинг раздач, профили и торрент-аплоад, но и более свежие механики: AI-поиск по раздачам, умный помощник загрузки, релиз-группы, социальную авторизацию и мультитрекерный импорт.

Документация ниже описывает текущее состояние репозитория в ветке `main` и рассчитана в первую очередь на локальный запуск через Docker.

## Что уже есть в проекте

- Полноценный торрент-трекер на PHP с MySQL и Memcached.
- Docker-окружение для локальной разработки: PHP 8.4, MySQL 8.4, phpMyAdmin, Memcached.
- Семанический AI-поиск в [browse.php](/browse.php), который ищет не только по точному названию, но и по `name`, `search_text`, `descr`, `tags`, комментариям и связанным текстам.
- Умный AI-помощник загрузки в [upload.php](/upload.php) и [film.php](/film.php), который:
  - нормализует название релиза;
  - пытается определить категорию, качество, год, жанры, теги, формат, перевод;
  - умеет подтягивать краткую справку из Wikipedia;
  - собирает черновик описания;
  - проверяет описание перед публикацией и показывает проблемы.
- Автоматическое расширение поискового индекса раздач при аплоаде, редактировании и добавлении комментариев.
- Релиз-группы с привязкой раздач к группе.
- Социальная авторизация через Telegram и Apple ID.
- Мультитрекерная загрузка `.torrent` с возможностью сохранить внешние announce и исходный `info_hash`.

## Стек и окружение

На текущий момент репозиторий ориентирован на такой стек:

- PHP `8.4` c Apache.
- MySQL `8.4`.
- Memcached.
- phpMyAdmin для локальной работы с базой.

Фактическая конфигурация берётся из:

- [Dockerfile](/Dockerfile)
- [docker-compose.yml](/docker-compose.yml)
- [include/secrets.php](/include/secrets.php)

Важно: по умолчанию [include/secrets.php](/include/secrets.php) настроен именно под Docker-сервисы `db` и `memcached`. Если вы запускаете проект без Docker, эти параметры нужно менять вручную.

## Быстрый старт через Docker

### 1. Клонирование

```bash
git clone https://github.com/nickmsk9/torrentside.ru.git
cd torrentside.ru
```

### 2. Запуск контейнеров

```bash
docker compose up -d --build
```

После старта будут доступны:

- сайт: `http://localhost:8084`
- phpMyAdmin: `http://localhost:8085`
- MySQL с хоста: `127.0.0.1:3308`

### 3. База данных

На чистом Docker-volume MySQL автоматически импортирует базовый дамп из:

- [torrent2.sql](/torrent2.sql)

Если база уже существует и вы обновляете старую установку, миграции из папки [sql](/sql) нужно применять вручную.

## Обязательные миграции для новых возможностей

Сейчас в репозитории есть минимум две важные SQL-миграции:

- [sql/2026-04-02-release-groups.sql](/sql/2026-04-02-release-groups.sql)
- [sql/2026-04-03-ai-search.sql](/sql/2026-04-03-ai-search.sql)

Пример применения из локальной машины:

```bash
mysql -h 127.0.0.1 -P 3308 -u root torrent2 < sql/2026-04-02-release-groups.sql
mysql -h 127.0.0.1 -P 3308 -u root torrent2 < sql/2026-04-03-ai-search.sql
```

Если вы применяете миграции внутри контейнера MySQL, команда будет другой, но смысл тот же: таблицы/поля для релиз-групп и fulltext-индекс AI-поиска должны быть добавлены в рабочую базу.

## AI-поиск по раздачам

Текущая логика поиска сделана как гибридный поиск по индексу трекера. На странице [browse.php](/browse.php) пользователь может вбивать:

- название фильма или игры;
- фразу из описания;
- имя актёра;
- фрагмент комментария;
- “естественный” запрос вроде `фильм про непогоду`.

Поиск учитывает:

- прямые совпадения по названию;
- описание раздачи;
- теги;
- расширенный `search_text`;
- данные, добавленные в индекс через комментарии и сопутствующий текст;
- опечатки, раскладку, транслит и альтернативные формулировки.

Для поддержки этой механики используется fulltext-индекс:

- [sql/2026-04-03-ai-search.sql](/sql/2026-04-03-ai-search.sql)

После обновления старой базы индекс нужно пересобрать:

```bash
docker compose exec -T php php tools/rebuild_torrent_search_index.php
```

Или с другим batch size:

```bash
docker compose exec -T php php tools/rebuild_torrent_search_index.php 500
```

Скрипт для пересборки лежит здесь:

- [tools/rebuild_torrent_search_index.php](/tools/rebuild_torrent_search_index.php)

## AI-помощник загрузки

AI-помощник загрузки сейчас встроен в:

- [upload.php](/upload.php)
- [film.php](/film.php)
- серверную обработку через [upload_assistant.php](/upload_assistant.php)
- основную логику через [include/upload_ai.php](/include/upload_ai.php)

Что он умеет на текущий момент:

- работать по одному названию релиза;
- дополнительно анализировать `.torrent`, `MediaInfo` и `NFO`;
- нормализовать “грязное” имя релиза;
- предлагать:
  - красивое название;
  - оригинальное название;
  - год;
  - жанры;
  - теги;
  - качество;
  - формат;
  - перевод;
  - примерную категорию;
  - черновик описания;
- выполнять базовую “AI-модерацию” описания:
  - слишком короткий текст;
  - отсутствующие обязательные блоки;
  - ошибки BBCode;
  - подозрительные внешние ссылки;
  - CAPS LOCK;
  - запрещённые формулировки.

Если разрешён внешний lookup, помощник может брать краткую справку из Wikipedia и использовать её как основу для описания и названий.

## Релиз-группы

Поддержка релиз-групп уже встроена в логику добавления и редактирования раздач. Структуры для этого создаются миграцией:

- [sql/2026-04-02-release-groups.sql](/sql/2026-04-02-release-groups.sql)

Что добавлено:

- таблица `release_groups`;
- участники групп;
- закладки групп;
- стена группы;
- поле `release_group_id` у раздач.

После применения миграции аплоадер может привязать раздачу к релиз-группе при загрузке.

## Социальная авторизация

В проекте поддерживаются:

- Telegram Login Widget
- Apple ID

Основные переменные читаются из [include/secrets.php](/include/secrets.php) через `getenv(...)`.

Пример переменных окружения:

```bash
SOCIAL_TELEGRAM_ENABLED=1
SOCIAL_TELEGRAM_BOT_USERNAME=your_bot_name
SOCIAL_TELEGRAM_BOT_TOKEN=123456:ABCDEF
SOCIAL_TELEGRAM_WIDGET_SIZE=large
SOCIAL_TELEGRAM_REQUEST_WRITE_ACCESS=0
SOCIAL_TELEGRAM_AUTH_TTL=900

SOCIAL_APPLE_ENABLED=1
SOCIAL_APPLE_CLIENT_ID=com.example.web
SOCIAL_APPLE_TEAM_ID=TEAMID1234
SOCIAL_APPLE_KEY_ID=KEYID1234
SOCIAL_APPLE_PRIVATE_KEY_PATH=/absolute/path/AuthKey_KEYID1234.p8
SOCIAL_APPLE_SCOPE="name email"
```

Практические замечания:

- Telegram требует настроенного бота и домена через `@BotFather`.
- Apple ID требует HTTPS и корректного `redirect_uri`.
- При первом успешном social login движок создаёт обычную запись пользователя и связывает её с провайдером.

## Важные замечания по эксплуатации

- Файл [include/secrets.php](/include/secrets.php) сейчас хранит параметры подключения в коде. Для production это лучше вынести в переменные окружения или отдельный приватный конфиг.
- Базовый импорт берётся из [torrent2.sql](/torrent2.sql). Если у вас уже есть рабочая база, не затирайте её повторным импортом — применяйте только нужные миграции.
- После включения AI-поиска на старой базе обязательно пересоберите поисковый индекс через [tools/rebuild_torrent_search_index.php](/tools/rebuild_torrent_search_index.php).
- Перед production-развёртыванием желательно проверить:
  - HTTPS;
  - реальные пароли для MySQL;
  - бэкапы базы;
  - доступы к директориям;
  - конфигурацию `SITEURL`, cookie и почтовых настроек.
